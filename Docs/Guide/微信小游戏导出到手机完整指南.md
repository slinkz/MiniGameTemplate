# 微信小游戏导出到手机完整指南

> 从 Unity 编辑器到手机上玩到游戏的完整链路  
> 最后更新：2026-04-24 | 适用版本：Unity 2021.3.17f1 + WX SDK 0.1.32

---

|---|------|------|------|

## 🚀 完整操作步骤

### 第一步：申请 AppID（如果还没有）

1. 打开 [微信公众平台](https://mp.weixin.qq.com)
2. 点击「立即注册」→ 选择「小游戏」
3. 按流程填写邮箱、密码、主体信息（个人即可）
4. 注册完成后进入管理后台
5. 左侧菜单 →「开发管理」→「开发设置」→ 复制 **AppID**

> 💡 个人开发者就能注册小游戏，不需要企业资质。即使基础信息没填完整，也能拿到 AppID 用于开发调试。

---

### 第二步：配置 Unity 项目（WX SDK 面板）

1. **打开转换面板**：Unity 菜单栏 →「微信小游戏」→「转换小游戏」

2. **填写关键配置**：

   | 配置项 | 当前值 | 建议值 | 说明 |
   |--------|--------|--------|------|
   | **游戏 AppID** | （空） | 填你的 AppID | 从公众平台复制 |
   | **项目名称** | （空） | `MiniGameTemplate` | 任意名称 |
   | **导出路径** | 项目根目录 | `C:/WXMiniGame/output` | ⚠️ 建议用**不含中文和空格**的独立目录 |
   | **首包资源加载方式** | CDN (0) | **小游戏包内** | 调试阶段选包内，不需要 CDN |
   | **游戏资源CDN** | （空） | 调试阶段可空 | 用包内加载就不需要 |
   | **内存大小** | 256MB | 256MB | 保持默认 |
   | **游戏方向** | 竖屏 (0) | 看你的游戏 | 横屏=1，竖屏=0 |

3. **编译选项**（当前已配置的比较合理）：
   - `DevelopBuild`: **建议开启**（调试阶段方便看错误）
   - `Il2CppOptimizeSize`: ✅ 已开启（减小包体）
   - `Webgl2`: 关闭（兼容性更好）
   - `DeleteStreamingAssets`: ✅ 已开启

---

### 第三步：在 Unity 中配置导出路径

在面板中，把「导出路径」（DST）设为一个干净的目录，比如：

```
C:/WXMiniGame/output
```

> ⚠️ **关键提醒**：
> - 路径不要放在 Unity 项目内部
> - 路径不要包含中文、空格或特殊字符
> - 确保磁盘空间充足（首次导出可能 200MB+）

---

### 第四步：构建并转换

1. 确认 **Build Settings** 中 Platform 已切换到 **WebGL**
   - 如果当前是 `StandaloneWindows64`，需要先 Switch Platform
   - Unity 菜单 → File → Build Settings → 选 WebGL → **Switch Platform**
   - ⚠️ 首次切换会重新导入资源，耗时较长（5-15分钟）

2. 确认 Build Settings 中 **Scenes In Build** 包含你要的场景：
   - ✅ `Assets/_Framework/GameLifecycle/Boot.unity`（启动场景）
   - ✅ 其他游戏场景

3. **回到 WX 转换面板**，点击 **「导出并转换」**（或「生成并转换」）
   - Unity 会先执行 WebGL Build
   - 然后 SDK 脚本自动将 WebGL 输出转换为微信小游戏格式
   - 首次构建时间较长（10-30分钟），耐心等待
   - 控制台会显示进度，结束后会弹出提示

4. 构建完成后，导出目录结构类似：
   ```
   C:/WXMiniGame/output/
   ├── game.js              # 游戏入口
   ├── game.json            # 小游戏配置
   ├── project.config.json  # 开发者工具项目配置
   ├── webgl/               # WebGL 资源
   ├── ...
   ```

---

### 第五步：安装并配置微信开发者工具

1. 下载 [微信开发者工具（稳定版）](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 安装并打开
3. 用**注册公众平台的微信号**扫码登录
4. 选择「小游戏」项目类型

---

### 第六步：在开发者工具中打开项目

1. 点击「+」号（导入项目）
2. **项目目录**：选择你的导出目录 `C:/WXMiniGame/output`
3. **AppID**：填你的小游戏 AppID（和 Unity 面板中一致）
4. 点击「导入」

导入成功后，开发者工具会：
- 自动编译小游戏项目
- 在左侧模拟器中显示运行效果
- 你已经可以在 **PC 模拟器** 中看到游戏了！

> 💡 如果模拟器报错，看右侧「控制台」面板的错误信息排查。

---

### 第七步：在手机上预览（重点！）

有**两种方式**可以在手机上看到效果：

#### 方式 A：预览模式（推荐，最快）

1. 开发者工具顶部工具栏 → 点击 **「预览」** 按钮
2. 等待编译上传（几十秒到几分钟）
3. 会生成一个 **二维码**
4. 用手机微信 **扫码**
5. 🎮 **手机上就能玩了！**

> ⚠️ 预览码有效期 **30 分钟**，过期需重新生成  
> ⚠️ 只有**你自己的微信**能扫，其他人扫无效

#### 方式 B：真机调试模式（带调试面板）

1. 开发者工具顶部 → 点击 **「真机调试」**
2. 扫码后手机上会多一个 **调试浮层**（vConsole）
3. 可以看到 console.log、网络请求、性能数据
4. PC 上的开发者工具也会同步显示手机端日志

> 💡 如果需要排查性能问题或 bug，用这个模式。

#### 方式 C：体验版（给其他人玩）

1. 开发者工具顶部 → 点击 **「上传」**
2. 填写版本号和备注 → 上传
3. 去微信公众平台 → 管理 → 版本管理 → 将该版本设为 **「体验版」**
4. 在「成员管理」中添加体验者的微信号
5. 体验者打开小游戏即可（通过体验版链接或扫码）

---

### 🎮 试玩版专用流程（Playable Ad）

如果你用的是「试玩转换工具面板」：

1. Unity 菜单 →「微信小游戏」→「转换小游戏试玩」
2. 配置导出路径（同样用独立目录）
3. 点击导出
4. 试玩版导出的是一个**单 HTML 文件**（或小型目录）
5. 可以直接在浏览器中打开测试
6. 也可以上传到广告平台作为试玩广告素材

> 试玩版不需要 AppID，也不需要开发者工具——它是独立的 HTML 产物。

## 🔧 常见问题排查

### Q1：Switch Platform 到 WebGL 后编译报错

```
确认 Unity Hub 中已安装 WebGL Build Support 模块
Unity Hub → Installs → 你的版本 → Add Modules → WebGL Build Support
```

### Q2：开发者工具报 "appid 不合法"

- 确认 AppID 正确且是「小游戏」类型（不是小程序）
- 确认开发者工具登录的微信号是该 AppID 的管理员或开发者

### Q3：开发者工具中白屏或黑屏

- 检查导出目录是否正确指向包含 `game.js` 的根目录
- 查看开发者工具控制台的报错信息
- 确认 `project.config.json` 中的 appid 与面板一致

### Q4：手机预览加载很慢

- 确认「首包资源加载方式」选了**小游戏包内**（不需要 CDN）
- 如果选了 CDN 但没配置 CDN 地址，会加载失败
- 首次预览需要上传编译产物，取决于网速

### Q5：预览二维码扫码后闪退

- 检查内存大小设置（256MB 通常够用）
- 在开发者工具中先用模拟器确认没有 JS 错误
- 尝试开启 `DevelopBuild` 获取更详细的错误信息

### Q6：纹理/字体缺失

- 确认 `DeleteStreamingAssets` 的行为符合预期
- 检查 `bundlePathIdentifier` 和 `bundleExcludeExtensions` 配置
- 可能需要调整资源打包策略

## 🎯 最小路径（从零到手机上玩，最快多久？）

假设你已有微信公众平台账号和 AppID：

1. ⏱️ **Switch Platform 到 WebGL** — 5~15 分钟（首次重导入）
2. ⏱️ **配置 WX SDK 面板** — 2 分钟
3. ⏱️ **导出并转换** — 10~30 分钟（首次构建）
4. ⏱️ **安装开发者工具 + 导入项目** — 5 分钟
5. ⏱️ **点预览 + 扫码** — 1~3 分钟

**总计：约 25~55 分钟**可以在手机上玩到。
